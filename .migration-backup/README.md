## ✨ Features

### Public site
- 🎨 **Black & white terminal aesthetic** — inspired by termux / starlink / x.com
- 🖼️ **Live profile picture** — uploaded from admin
- 🚀 **Hero section** with greeting prefix (`~/darrell`), tagline, CV download
- 📊 **Skills with proficiency bars** — frontend, backend, database, languages (configurable categories)
- 💼 **Featured projects** with optional images, tech stack icons, case-study expand, **Live Demo** & **Source Code** buttons
- 🔧 **What I Build** — purpose-driven build cards
- 🛠️ **Currently Building** — what's in flight right now
- 🗺️ **Journey timeline** — milestones with highlights
- 📈 **GitHub activity graph** — driven by your username
- ✍️ **Blog** — markdown posts, slugs, tags, cover images, reader page at `/blog/:slug`
- 📬 **Working contact form** — saves to DB **and** emails you via Resend
- 🌀 **Stack marquee** — auto-scrolling tech ribbon
- 🏢 **Synapex section** — startup story
- 🦶 **Editable footer** with section links & social icons

### Admin CMS (`/admin`)
Every piece of content is editable, no redeploy needed.

| Section | What you can edit |
|---|---|
| **Settings** | Name, tagline, email, phone, location, CV URL, GitHub username, profile picture |
| **Skills** | Category, name, icon, proficiency % (slider) |
| **Projects** | Title, description, tech, case study (problem → result), image, demo URL, source URL, visibility toggle |
| **What I Build** | Purpose-driven build cards |
| **Currently Building** | In-flight work |
| **Milestones** | Year, title, description, highlight flag |
| **Blog** | Title, slug, excerpt, markdown content, cover image, tags, publish toggle |
| **Social Links** | Platform, URL, icon |
| **Footer** | Sectioned link groups |
| **Inbox** | View contact form messages |

### Security
- Role-based access via dedicated `user_roles` table (no privilege escalation)
- `has_role()` SECURITY DEFINER function prevents recursive RLS
- All public tables have RLS — public reads, admin-only writes
- Contact submissions accept anonymous inserts but only admins can read

## 🧱 Tech Stack

- **Framework**: React 18 + Vite 5 + TypeScript 5
- **Styling**: Tailwind CSS 3 + semantic design tokens (HSL)
- **Animation**: Framer Motion
- **Icons**: Lucide React + Simple Icons CDN (brand colors)
- **State / Data**: TanStack Query
- **Routing**: React Router v6
- **Backend**: Lovable Cloud (Supabase) — Postgres, Auth, Storage, Edge Functions
- **Email**: Resend (via Supabase Edge Function)
- **UI primitives**: shadcn/ui (Radix)

## 🚀 Getting Started

### Prerequisites
- Node 18+ or Bun
- A Lovable account (Cloud auto-provisioned)

### Install & run
```bash
bun install
bun dev
```
Open http://localhost:5173

### Environment
The `.env` file is auto-generated by Lovable Cloud and contains:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`
- `VITE_SUPABASE_PROJECT_ID`

Backend secrets (set in Cloud, never in code):
- `RESEND_API_KEY` — your Resend API key
- `CONTACT_NOTIFICATION_EMAIL` — where contact-form messages are sent

## 🔐 Admin Access

1. Visit **`/admin/login`**
2. Click **"Need an account? Sign up"**
3. Register with **darrelmucheri@gmail.com** — a database trigger automatically grants the `admin` role to this email
4. You're in. Edit anything.

To grant admin to a different email, update the `assign_admin_on_signup` trigger or insert directly into `user_roles`.

---

## 📨 Contact Form Setup

The form posts to the `send-contact-email` edge function which:
1. Validates input (email format, length caps)
2. Inserts into `contact_messages` (visible in admin inbox)
3. Emails you via Resend (using `onboarding@resend.dev` until you verify a domain)

To send from your own domain, verify it at [resend.com/domains](https://resend.com/domains) and update the `from` field in `supabase/functions/send-contact-email/index.ts`.

## 🧩 Adding Content

Use the admin panel for everything. Some quick notes:

- **Project images / blog covers / profile pic** — uploaded via the `ImageUpload` component to the `site-media` Supabase bucket (public read).
- **Tech stack icons** — type the lowercase tech name (e.g. `react`, `typescript`, `postgresql`). The `TechIcon` component pulls colored brand icons from the Simple Icons CDN. See `src/lib/tech-icons.tsx` for the full map.
- **Lucide icons** — for milestones / what-i-build / social links, use a slug (e.g. `bot`, `rocket`, `github`). See `src/lib/icons.ts`.
